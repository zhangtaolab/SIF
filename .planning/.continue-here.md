# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [x] **CONSTRAINT: SQLite DDL triggers implicit commit** — Creating tables inside a transaction that later rolls back will NOT roll back the DDL. Test assertions must pre-create tables outside the transaction-under-test. — Structural mitigation: never assert DDL rollback in SQLite tests.
- [x] **CONSTRAINT: macOS load_extension("vec0") fails** — The sqlite-vec C extension cannot be loaded via raw `load_extension()` on macOS. Use `sqlite_vec.load(conn)` from the Python package instead.
- [x] **CONSTRAINT: Qwen3-Reranker-0.6B is NOT a CrossEncoder** — Using `sentence_transformers.CrossEncoder` with this model causes "cannot reshape tensor of 0 elements". It requires `AutoModelForCausalLM` with yes/no token logit prediction.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Stale test files referencing removed APIs | Old test files import symbols that no longer exist (Chunk, ChunkingStrategy, ParseResult, index_add, search_query, etc.) | blocking | Run pytest collection after any API rename/removal. Delete or fix stale tests immediately. |
| DatabaseConnection.execute without commit | SQLite's default isolation level requires explicit commit(). execute() returning a cursor from a closed connection yields ProgrammingError. | blocking | All write operations in DatabaseConnection helper methods must commit before returning. |
| Ruff formatting all files at once | Formatting 100+ files creates a huge uncommitted diff that obscures real changes. | advisory | Run ruff format only on files you modified, or commit functional changes before formatting. |
