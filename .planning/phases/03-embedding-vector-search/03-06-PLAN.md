       ```python
       settings = get_settings()
       try:
           manager = EmbeddingManager.from_settings(settings)
       except ImportError as e:
           console.print(f"[red]Embedding backend not installed: {e}[/red]")
           return
       except Exception as e:
           console.print(f"[red]Failed to load embedding model: {e}[/red]")
           return
       ```

    4. Inside the collection loop, replace the per-document embedding loop with cross-document batching:
       ```python
       for coll in collections:
           if not coll:
               continue
           console.print(f"\n[bold]Embedding collection: {coll.name}[/bold]")
           documents = doc_repo.list_by_collection(coll.id)
           chunker = create_chunker(chunk_strategy)

           # Collect all chunks across documents
           all_chunks: list[tuple[DocumentChunk, str]] = []  # (chunk, document_id)
           doc_chunks_map: dict[str, list[DocumentChunk]] = {}

           for doc in documents:
               chunk_repo.delete_by_document(doc.id)
               chunks = chunker.chunk(doc.content)
               doc_chunks_map[doc.id] = chunks
               for i, chunk in enumerate(chunks):
                   chunk.document_id = doc.id
                   chunk.sequence = i
                   all_chunks.append((chunk, doc.id))

           # Batch embed all chunks
           if all_chunks:
               try:
                   chunk_texts = [c.content for c, _ in all_chunks]
                   embedding_response = manager.embed(chunk_texts)
                   embeddings = embedding_response.embeddings

                   # Persist chunks and embeddings
                   batch_items: list[tuple[str, str, str | None, list[float]]] = []
                   for (chunk, doc_id), embedding in zip(all_chunks, embeddings):
                       chunk_repo.create(chunk)
                       batch_items.append((chunk.id, doc_id, chunk.id, embedding))

                   if batch_items:
                       vector_searcher = VectorSearcher(db.connection, manager.embed_single("probe").__len__())
                       vector_searcher.add_embeddings_batch(batch_items)

                   total_chunks += len(all_chunks)
               except Exception as e:
                   console.print(f"  [red]Error embedding collection {coll.name}: {e}[/red]")

           # Update collection stats
           documents = doc_repo.list_by_collection(coll.id)
           coll.chunk_count = sum(
               len(chunk_repo.get_by_document(d.id)) for d in documents
           )
           coll_repo.update(coll)
       ```

    5. Remove the old per-document loop entirely.

    6. Ensure `model_name = model or settings.model_name` is still computed but only used for display/logging if needed. Actually, since EmbeddingManager handles model_name internally, you can remove `model_name` from the local embedder creation but keep the `--model` CLI option for future use or pass it to settings override. For now, if `model` is provided, we should ideally override settings.model_name. Simpler approach: keep the option but don't use it directly for embedder creation since EmbeddingManager reads from settings. To respect the CLI option, temporarily override:

       Actually, a better approach: create a fresh Settings instance if model is provided:
       ```python

       if model:
           # Bypass cache for this call
           from docsift.config.settings import Settings
           settings = Settings(**settings.model_dump(), model_name=model)
       ```
       But this may be complex. For this task, simply keep the `model` parameter and if provided, pass it to EmbeddingConfig manually by creating a custom manager instead of from_settings. Let's do:

       ```python
       if model:
       ```
