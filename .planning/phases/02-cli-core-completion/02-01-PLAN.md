    Replace the body of `multi_get_cmd` in `src/docsift/cli/commands/get.py` with the following behavior:

    1. Keep the existing early-exit for missing index and the `fnmatch` import.
    2. Auto-detect the input type using D-01 priority (comma > glob > single):
       - If `","` is in `pattern`: split by commas, strip whitespace, filter out empty strings. For each item, try `doc_repo.get_by_id(item)`; if not found, iterate all collections and try `doc_repo.get_by_path(item, coll.id)`. Collect all found documents (preserve order, allow duplicates from different lookups but deduplicate by doc.id).
       - Else if `"*"` or `"?"` is in `pattern`: iterate all collections via `coll_repo.list_all()`, then all docs via `doc_repo.list_by_collection(coll.id)`. Match against `doc.path` or `doc.filename` using `fnmatch.fnmatch`. Collect matches.
       - Else (single item): try `doc_repo.get_by_id(pattern)`; if not found, iterate all collections and try `doc_repo.get_by_path(pattern, coll.id)`. Return the single found doc or empty list.
    3. If no documents found, print: `[yellow]No documents matching pattern: {pattern}[/yellow]` and return.
    4. If documents found, print: `[green]Found {len(matched_docs)} matching document(s)[/green]` followed by a blank line.
    5. For each matched document, print `[bold cyan]=== {doc.path} ===[/bold cyan]`, then the content truncated to `max_bytes` (keep existing truncation logic), then a blank line.

    Do NOT change the command signature or Click decorators.
  </action>
