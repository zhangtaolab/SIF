# Current buggy code

## PathContext dataclass (core/models.py:167-188)
Lacks `context_type` field. `_row_to_context()` in repositories.py reads `row["context_type"]` but cannot store it.

## ContextRepository.create() (repositories.py:345-355)
Hardcodes `'path'` in INSERT SQL:
```python
INSERT INTO contexts (id, target_id, context_type, content, created_at, updated_at)
VALUES (?, ?, 'path', ?, ?, ?)
```

## context_add() (cli/commands/context.py:59-71)
Never passes `type` to `repo.create()`:
```python
path_context = PathContext(path=actual_target, context=content)
repo.create(path_context)
```

## context_list() (cli/commands/context.py:127-132)
Hardcodes `"path"` in table row:
```python
table.add_row("path", ctx_item.path, content_text)
```

## _row_to_context() (repositories.py:416-425)
Reads `target_id` and `content` but not `context_type` into PathContext.
