# Current buggy code (same pattern in all three files)

## bm25.py _attach_contexts() (lines 82-96)
```python
def _attach_contexts(self, results: list[SearchResult]) -> list[SearchResult]:
    if not results:
        return results
    paths = list({r.path for r in results})
    placeholders = ", ".join(["?"] * len(paths))
    sql = f"""
        SELECT target_id, content FROM contexts
        WHERE context_type = 'path' AND target_id IN ({placeholders})
    """
    cursor = self.db.execute(sql, paths)
    context_map = {row["target_id"]: row["content"] for row in cursor.fetchall()}
    for result in results:
        result.context_description = context_map.get(result.path)
    return results
```

## vector.py _attach_contexts() (lines 103-117)
Identical pattern — literal match on `target_id = path`.

## hybrid.py _attach_contexts() (lines 84-117)
Identical pattern with MagicMock-safe row handling.

# Root cause
macOS resolves /tmp to /private/tmp. When a document is indexed, its path is stored as /private/tmp/... (resolved). When a user adds a path context, they type /tmp/... (unresolved). The contexts table stores /tmp/... but documents table stores /private/tmp/... The IN clause and dict lookup fail because the strings differ.

# Fix strategy
Normalize both sides using `os.path.realpath()` before comparison. The DB query should still use the original paths (to avoid changing the SQL), but the lookup should normalize.

             sql = f"""
                 SELECT target_id, content FROM contexts
                 WHERE context_type = 'path' AND target_id IN ({placeholders})
             """
             cursor = self.db.execute(sql, paths)
             rows = cursor.fetchall()
             # Handle both sqlite3.Row (dict-like) and plain tuple rows
             context_map: dict[str, str] = {}
             for row in rows:
                 if hasattr(row, "keys") and callable(getattr(row, "keys", None)):
                     try:
                         context_map[os.path.realpath(row["target_id"])] = row["content"]
                         continue
                     except (KeyError, TypeError):

                 # Fallback: assume tuple-like access (skip MagicMock rows in tests)
                 try:

                     val = row[1]
                     if not isinstance(key, str):

             for result in results:
                 normalized_path = os.path.realpath(result.path)
