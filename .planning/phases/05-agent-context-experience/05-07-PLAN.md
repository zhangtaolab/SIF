# Current buggy code

## main.py --index option (lines 27-33)
```python
@click.option(
    "--index",
    "-i",
    type=click.Path(),
    default=str(DEFAULT_INDEX_PATH),
    help="Path to the index database",
)
```
The `default=str(DEFAULT_INDEX_PATH)` bypasses Pydantic Settings. If DOCSIFT_DB_PATH is set, the CLI still uses the hardcoded default.

## main.py cli() function (lines 44-65)
```python
def cli(ctx: click.Context, index: str, config: str, verbose: bool, quiet: bool) -> None:
    ...
    ctx.obj["index_path"] = Path(index).expanduser()
```
The index_path is set from the Click option, which has the hardcoded default.

## main.py status_cmd (lines 90-124)
```python
@cli.command("status")
@click.pass_context
def status_cmd(ctx: click.Context) -> None:
    index_path = ctx.obj["index_path"]
    ...
```
Uses ctx.obj["index_path"] directly, which comes from the hardcoded default.

## Settings (settings.py:188-197)
```python
def get_db_path(self) -> Path:
    if self.db_path:
        return self.db_path
    from platformdirs import user_data_dir
    data_dir = Path(user_data_dir(self.app_name))
    data_dir.mkdir(parents=True, exist_ok=True)
    return data_dir / "docsift.db"
```
Settings respects DOCSIFT_DB_PATH via the db_path field.

# Fix strategy
Option A: Change the --index default to use Settings.get_db_path(). This affects ALL commands.
Option B: Only fix status_cmd to use Settings.get_db_path() when --index was not explicitly provided.

We choose Option A because it's the correct behavior — the CLI should default to the Settings-resolved path, not a hardcoded one. The --index option should still override when explicitly provided.
