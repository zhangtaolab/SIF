    Make these changes to `src/docsift/mcp/server.py`:

    1. Replace the `Database` import with `DatabaseConnection`:
       ```python
       from docsift.database.connection import DatabaseConnection
       ```
       Remove `from docsift.database.database import Database`.

    2. Remove the `db` attribute and `initialize` method from `MCPServer`. The class should only store `index_path`:
       ```python
       class MCPServer:
           def __init__(self, index_path: str) -> None:
               self.index_path = index_path
       ```

    3. In `_tool_query`, replace `with self.db.connection:` with `with DatabaseConnection(self.index_path).transaction() as conn:` and update all references from `self.db.connection` to `conn` inside that block. Also add schema initialization at the top of the block:
       ```python
       from docsift.database.schema import SchemaManager
       SchemaManager(conn).create_all()
       ```

    4. Do the same replacement in `_tool_lex_search`, `_tool_get`, `_tool_multi_get`. Use `DatabaseConnection(self.index_path).transaction() as conn:` and replace `self.db.connection` with `conn`. Add `SchemaManager(conn).create_all()` in each.

    5. In `_tool_status`, replace the `self.db` check and `self.db.get_stats()` with:
       ```python
       with DatabaseConnection(self.index_path).transaction() as conn:
           from docsift.database.schema import SchemaManager
           SchemaManager(conn).create_all()
           stats = SchemaManager(conn).get_stats()
       ```
       Remove the `if not self.db:` guard.

    6. In `_tool_vec_search`, keep the current fallback to `_tool_lex_search` but ensure it uses the new connection pattern (it delegates to `_tool_lex_search` which will already be updated).

    7. In `run_stdio_server`, remove `server.initialize()`.

    Ensure all repository and searcher instantiations inside the handlers use `conn` instead of `self.db.connection`.
