<objective>
Improve SQLite connection management to avoid race conditions in async and multi-threaded contexts per D-04 and D-05.

Purpose: `Database` caches a single connection with `check_same_thread=False`, which is unsafe across threads. `DatabaseConnection` and `ConnectionPool` also use `check_same_thread=False` and do not load `sqlite-vec` consistently. The MCP HTTP transport does not demonstrate safe connection usage.
Output: `Database` uses `sqlite_vec.load()` for extension loading; `DatabaseConnection` and `ConnectionPool` are thread-safe; `HTTPTransport` creates a fresh connection per request.
</objective>

       - Update `_enable_extensions()` to use `sqlite_vec.load(self._connection)` instead of the manual path-probing loop. Add `import sqlite_vec` at the top of the file if not already present. The method should look like:
         ```python
         def _enable_extensions(self) -> None:
             """Enable SQLite extensions."""
             self._connection.execute("PRAGMA foreign_keys = ON")
             try:
                 sqlite_vec.load(self._connection)
             except Exception:

         ```
       - Add a docstring to the `Database` class clarifying its intended use:
         ```python
         class Database:
             """Main database class for DocSift.
             
             This class caches a single connection and is intended for
             single-threaded CLI use via `transaction()`. Async or
             multi-threaded contexts should use `DatabaseConnection`.
             """
         ```
       - Keep `check_same_thread=False` in the `connection` property because CLI is single-threaded and this avoids false positives from Click's context handling.

    2. In `src/docsift/database/connection.py`:
       - Remove `check_same_thread=False` from `DatabaseConnection._create_connection()`. The connection should be created with the default `check_same_thread=True` behavior.
       - Ensure `_create_connection()` already loads `sqlite_vec.load(conn)`, sets `row_factory = sqlite3.Row`, and executes `PRAGMA foreign_keys = ON`. Verify these are all present and correct.
       - Update `ConnectionPool.get_connection()` to also:
         a. Remove `check_same_thread=False`.
         b. Load `sqlite_vec.load(conn)` after creating the connection.
         c. Set `conn.row_factory = sqlite3.Row`.
         d. Execute `conn.execute("PRAGMA foreign_keys = ON")`.
         The method should look like:
         ```python
         def get_connection(self) -> sqlite3.Connection:
             """Get a connection from the pool or create a new one."""
             if self._connections:
                 return self._connections.pop()
             
             conn = sqlite3.connect(str(self._db_path))
             conn.row_factory = sqlite3.Row
             conn.execute("PRAGMA foreign_keys = ON")
             try:
                 conn.enable_load_extension(True)
                 sqlite_vec.load(conn)
                 conn.enable_load_extension(False)
             except Exception:
                 pass
             return conn
         ```
